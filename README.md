## 简介

paper-cowork 是一个 **Claude Code 插件**，专门帮助本科毕业论文写作。它不是一个 AI 写作工具，而是一个 **写作流程助手**——帮你结构化地从选题走到终稿。

### 与传统 AI 写论文的区别

| 传统 AI 写论文 | paper-cowork |
|---------------|--------------|
| 一次生成全文，质量不可控 | **逐章推进**，每章你审阅通过才继续 |
| AI 瞎编参考文献 | **只引用你写的文献笔记**，不凭空编造 |
| AI 捏造实验数据 | **你提供什么数据就用什么**，缺的标注⚠️ |
| 改起来麻烦 | **断点续写 + 版本追踪**，随时退出恢复 |
| 千篇一律的风格 | **4 大类论文结构**自动适配 |

## 功能一览

| 阶段 | 做什么 | 产出 |
|------|--------|------|
| **主题确认** | 问专业、类型、选题，收敛为 1-2 个候选题目 | 论文蓝图 |
| **文献调研** | 整理你的文献笔记，生成检索关键词，补充英文文献 | 文献综述骨架 |
| **参考论文分析** | 分析优秀论文的写作结构和技巧 | 写作技巧汇总 |
| **素材收集** | 逐项问你有什么数据，检测文件是否存在 | 素材清单 |
| **论文写作** | 逐章起草 → 你审阅 → 修改 → 确认 | 初稿 |
| **格式输出** | GB/T 7714 引用、图表检查、降重、.docx 生成 | 论文终稿.docx |
| **修改追踪** | 处理导师反馈，归档每个版本 | 修改记录 |

## 快速开始

### 安装

```bash
# 克隆仓库
git clone https://github.com/Jonoo-no/paper-cowork.git
cd paper-cowork

# 临时加载（推荐先试用）
claude --plugin-dir ./
```

### 第一次使用

在 Claude Code 中依次运行两个 slash command：

```
第 1 步：/启动论文工作区    ← 一次性初始化
  → 问工作区路径
  → 创建目录结构、放入模板
  → 检测已安装的辅助技能
  → 引导你放入材料

第 2 步：/论文流水线         ← 每次写论文时运行
  → 从上次进度继续
  → 逐阶段推进（当前 Phase 1→7）
  → 每一阶段结束问你「继续吗」
```

> 💡 每次退出 Claude 后，重新打开时只需要再次运行 `/论文流水线`，它会自动从断点继续。

## 论文类型支持

插件根据你的专业自动匹配章节结构：

| 大类 | 包含专业 | 默认章节 | 特点 |
|------|---------|---------|------|
| 🛠 **理工科** | 计算机、电子、机械、土木、自动化 | 绪论 → 相关技术 → 系统设计 → 实验 → 总结 | 有架构图、代码片段、实验结果 |
| 🔬 **理科/医学** | 数学、物理、化学、生物、临床 | 引言 → 材料方法 → 结果 → 讨论 → 结论 | 有实验数据、统计图表、病例资料 |
| 📖 **文科** | 中文、历史、哲学、外语、新闻 | 绪论 → 理论基础 → 主体分析 → 结语 | 以文本分析、理论辨析为主 |
| 📊 **经管/社科** | 经济、管理、法学、教育、社会学 | 绪论 → 理论基础 → 现状分析 → 实证分析 → 结论 | 有问卷数据、回归模型、政策文件 |

> 以上结构可以自由增删改动，AI 会按你的要求调整。

## 引用规范

**插件不会凭空生成参考文献。** 所有引用都来自你放在 `文献笔记/` 目录下的笔记文件。

```
你读论文 → 你写笔记（放文献笔记/） → AI 引用
          ↳ 包含标题、作者、年份、期刊
          ↳ 包含可引用的具体观点
          ↳ 包含与你论文的关系
```

**引用格式：** GB/T 7714（中国国家标准）

```
[1] 作者1, 作者2. 论文标题[J]. 期刊名, 年份, 卷(期): 起止页码.
[2] 作者. 专著书名[M]. 出版地: 出版社, 年份.
```

## 完整工作流程

### Phase 1：论文主题确认

```
① 问专业 → 映射到 4 大类
② 问论文类型（综述型 / 实践型 / 实证型）
③ 问字数要求
④ 对话收敛题目
⑤ 输出 01-论文蓝图.md
```

### Phase 2：文献调研

```
① 检测文献笔记/ 目录已有内容
② 生成中英文检索关键词
③ 中文文献 → 告知从知网导出 .ris 的方法
④ 英文文献 → 可选自动检索补充
⑤ 读取已有笔记，整理分类
⑥ 输出 02-文献综述骨架.md
```

### Phase 3：参考论文分析

```
① 读取参考论文案例/ 目录
② 按 6 个维度分析每篇论文的结构和技巧
③ 汇总写作技巧清单
```

### Phase 4：数据与素材收集

```
① 根据论文大类逐项询问材料
② 检测文件是否真实存在
③ 缺失 → 标注 ⚠️，不虚构
④ 输出 03-素材清单.md
```

### Phase 5：论文写作

```
逐章推进（每章：起草 → 你审阅 → 修改 → 确认）
最后写 Abstract，最后定 Title
输出 → 05-初稿/
```

### Phase 6：格式 + 引用 + 降重

```
① GB/T 7714 参考文献格式
② 图表编号自查
③ 内容自查（⚠️ 残留标记、数字格式）
④ 降重（可选，推荐中等程度）
⑤ 输出 论文终稿.docx
```

### Phase 7：修改追踪

```
① 你贴导师意见
② AI 生成修改方案
③ 你确认 → 执行修改
④ 归档版本（v2/v3...）
输出 → 06-修改稿/
```

## 常见问题

### 没有 Obsidian 能用吗？

可以。插件使用普通文件夹，不需要 Obsidian。运行 `/启动论文工作区` 时选择「没有知识库」即可。

### 学校没有发 .docx 模板怎么办？

Phase 6 会输出默认格式的 Markdown 文件，你在 Word 中手动套模板即可。

### 写到一半退出，进度会丢吗？

不会。每次进度都保存在 `.pipeline-state.json` 中，下次运行 `/论文流水线` 自动从断点继续。

### 插件装的技能没安装全，能用吗？

能。每个可选 skill 缺失时对应功能会跳过，不影响主流程。详见 [INSTALL.md](./INSTALL.md)。

### 导师修改了几轮后，能追溯吗？

可以。Phase 7 会归档每次修改为独立版本（v2、v3...），附带修改说明。
