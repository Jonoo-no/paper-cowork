# paper-cowork 🎓

**本科毕业论文写作助手** — 一个 Claude Code 插件，从选题到终稿的全流程辅助。

> 📌 适用于：本科毕业论文（中文）、综述型/实践型/实证型论文
> 📌 不依赖 Obsidian，使用普通文件夹即可

---

## 功能一览

| 阶段 | 做什么 |
|------|--------|
| **主题确认** | 帮你收敛选题，确定论文类型和结构 |
| **文献调研** | 整理你的文献笔记，补充英文检索 |
| **参考论文分析** | 分析优秀论文的写作结构和技巧 |
| **素材收集** | 引导你提供实验数据、流程图等材料 |
| **论文写作** | 逐章起草 → 你审阅 → 修改，引用仅来自你的笔记 |
| **格式输出** | 套用学校模板生成 .docx，GB/T 7714 参考文献 |
| **修改追踪** | 处理导师反馈，归档版本 |

## 快速开始

### 安装

```bash
# 方式 1：克隆后临时加载（推荐先试用）
git clone https://github.com/你的用户名/paper-cowork.git
cd paper-cowork
claude --plugin-dir ./

# 方式 2：永久安装
# 把 agents/ 目录复制到 .claude/agents/ 下
cp -r agents ~/.claude/agents/paper-cowork/

# 方式 3：Windows 用户双击 install.bat
```

### 可选依赖

```bash
# 用于自动格式化 .docx 输出
pip install python-docx

# 推荐安装的辅助 Claude Code Skills（非必须）：
# - nature-figure：数据图表生成
# - aigc-reduce：论文降重
# - nature-academic-search：英文文献检索
```

### 使用

```
第 1 步：在 Claude Code 中运行  /启动论文工作区
         → 一次性初始化目录、模板、依赖检测

第 2 步：运行  /论文流水线
         → 按 Phase 1→7 逐步推进，每次自动从断点继续
```

## 工作区结构

运行 `/启动论文工作区` 后，会在你指定的位置创建以下结构：

```
workspace/
├── 论文模板/                  ← 放学校下发的 .docx 模板
│
├── 参考论文案例/               ← 放参考论文 PDF 或分析笔记
│   └── 参考论文分析模板.md
│
├── 文献笔记/                  ← 放你读过的论文笔记
│   └── 文献笔记模板.md
│
├── 素材/                     ← 实验数据、截图、表格等
│   ├── 实验数据/
│   └── 流程图/
│
└── 论文项目/
    └── 毕业论文/
        ├── .pipeline-state.json   ← 进度状态（自动管理）
        ├── 01-论文蓝图.md          ← Phase 1 产出
        ├── 02-文献综述骨架.md       ← Phase 2 产出
        ├── 03-素材清单.md           ← Phase 4 产出
        ├── 05-初稿/                ← Phase 5 产出
        └── 06-修改稿/              ← Phase 7 产出
```

## 工作流程详解

```
/启动论文工作区（一次性）
    │
    ▼
/论文流水线（反复运行，断点续传）
    │
    ├─ Phase 1：论文主题确认  →  01-论文蓝图.md
    │     · 问专业、论文类型、字数要求
    │     · 收敛选题，确定章节结构（4 大类自动适配）
    │
    ├─ Phase 2：文献调研      →  02-文献综述骨架.md
    │     · 生成检索关键词（中英文）
    │     · 读取你的文献笔记，整理分类
    │     · 可选：自动补充英文文献
    │
    ├─ Phase 3：参考论文分析  →  写作技巧汇总
    │     · 分析你放入的参考论文结构和技巧
    │     · 输出可借鉴的写作方法
    │
    ├─ Phase 4：数据与素材收集 → 03-素材清单.md
    │     · 根据论文类型逐项询问所需材料
    │     · 检测文件是否真实存在
    │
    ├─ Phase 5：论文写作      →  05-初稿/
    │     · 逐章起草 → 你审阅 → 修改 → 确认
    │     · 引用仅来自你的文献笔记
    │     · 数据仅来自你的素材清单
    │     · Abstract 最后写
    │
    ├─ Phase 6：格式+引用+降重 → 论文终稿.docx
    │     · GB/T 7714 参考文献格式
    │     · 图表编号自查
    │     · 可选：aigc-reduce 中等程度降重
    │     · 输出 .docx（套用学校模板）
    │
    └─ Phase 7：修改追踪     →  06-修改稿/v2/
          · 处理导师反馈，归档各版本
```

## 论文类型与章节结构

插件会根据你的专业自动适配章节结构：

| 大类 | 包含专业 | 默认章节结构 |
|------|---------|-------------|
| 🛠 **理工科** | 计算机、电子、机械、土木 | 绪论 → 相关技术 → 系统设计 → 实验 → 总结 |
| 🔬 **理科/医学** | 数学、物理、化学、生物、临床 | 引言 → 材料方法 → 结果 → 讨论 → 结论 |
| 📖 **文科** | 中文、历史、哲学、外语 | 绪论 → 理论基础 → 主体分析 → 结语 |
| 📊 **经管/社科** | 经济、管理、法学、教育 | 绪论 → 理论基础 → 现状分析 → 实证分析 → 结论 |

每个章节结构都可以自由修改。

## 引用规范

插件**不会凭空生成参考文献**。所有引用来源：

1. **你的文献笔记**（`文献笔记/` 目录下的 `.md` 文件）
2. **你在知网等平台搜索后整理的笔记**
3. **英文文献补充**（可选安装 nature-academic-search 后自动检索）

参考文献格式：GB/T 7714（中国国家标准）

## 常见问题

### Q: 没有 Obsidian 能用吗？
可以。插件使用普通文件夹，不需要 Obsidian。创建时选「没有知识库」即可。

### Q: 学校没有发 .docx 模板怎么办？
插件会使用默认格式输出。你可以在 Phase 6 时手动在 Word 中套模板。

### Q: 写到一半退出，进度会丢吗？
不会。每次进度都保存在 `.pipeline-state.json` 中，下次运行 `/论文流水线` 自动从断点继续。

### Q: 插件装的技能没安装全，能用吗？
能。每个可选 skill 缺失时对应功能会跳过，不影响主流程。

### Q: 导师修改了几轮后，能追溯吗？
可以。Phase 7 会归档每次修改为独立版本（v2、v3...），并附带修改说明。

- **插件格式**：Claude Code Plugin（`plugin.json` + `agents/`）
