# 启动论文工作区

你正在运行 paper-cowork 插件的初始化流程。目标是为用户创建一个结构化的论文工作区。严格执行以下步骤，无法继续的步骤用占位符替代并标注。

---

## Step 1: 欢迎与说明

向用户说明：

> 欢迎使用 **paper-cowork 论文写作插件**！
>
> 这个插件帮助你完成本科毕业论文从选题到终稿的全流程。
> 现在先做一次性初始化——创建工作区目录结构、放入模板、检测可用的辅助技能。
>
> 整个过程约 5 分钟，后续写论文时只需运行 `/论文流水线`。

---

## Step 2: 确定工作区路径

问用户两个问题并等待回答：

1. **「你有没有 Obsidian 知识库？如果有，路径是什么？」**
   - 回答「有」→ 记录路径，后续在知识库内创建工作区
   - 回答「没有」或「不用 Obsidian」→ 使用普通文件夹

2. **「工作区想创建在哪里？默认在当前目录下的 `paper-workspace/`」**
   - 用户可以输入自定义路径，或确认使用默认路径

获取后记为 `workspace_path`（下文使用）。

---

## Step 3: 创建工作区目录结构

在 `workspace_path` 下创建：

```
{workspace_path}/
├── 论文模板/
├── 参考论文案例/
├── 文献笔记/
├── 素材/
│   ├── 实验数据/
│   └── 流程图/
└── 论文项目/
    └── 毕业论文/
        ├── 05-初稿/
        └── 06-修改稿/
```

创建完成后向用户确认：「目录结构已创建。」

---

## Step 4: 复制模板文件

从 `${CLAUDE_PLUGIN_ROOT}/templates/` 复制以下文件到工作区对应位置：

| 源 | 目标 |
|----|------|
| `论文蓝图模板.md` | `{workspace_path}/论文项目/毕业论文/01-论文蓝图.md` |
| `文献笔记模板.md` | `{workspace_path}/文献笔记/文献笔记模板.md` |
| `参考论文分析模板.md` | `{workspace_path}/参考论文案例/参考论文分析模板.md` |
| `素材清单模板.md` | `{workspace_path}/论文项目/毕业论文/03-素材清单.md` |

告诉用户：「模板文件已放入对应文件夹。」

---

## Step 5: 检测依赖 skill

扫描 `.claude/agents/` 目录，检查以下 skill 是否存在：

| Skill | 类别 | 用途 |
|-------|------|------|
| `nature-academic-search` | ⭐ 推荐 | 英文文献自动检索（Phase 2） |
| `nature-figure` | ⭐ 推荐 | 数据图表生成（Phase 5） |
| `aigc-reduce` | ⭐ 推荐 | 论文降重（Phase 6） |
| `nature-polishing` | 🔹 可选 | 英文摘要润色 |
| `nature-paper2ppt` | 🔹 可选 | 答辩 PPT 生成 |

输出检测报告，格式如：

```
📋 依赖检测报告
✅ nature-academic-search — 已安装
❌ nature-figure — 未安装（图表生成将跳过）
❌ aigc-reduce — 未安装（降重功能不可用）
...
```

然后**逐个询问缺失的推荐 skill**：「xxx 未安装，要安装吗？（推荐）」
- 要 → 给出安装方式（提供仓库地址或命令）
- 不需要 → 标注"跳过"

---

## Step 6: 引导用户放入材料

向用户说明每个目录的用途：

```
📂 论文模板/
   放学校下发的 .docx 论文模板（用于最终格式输出）

📂 参考论文案例/
   放你找到的优秀参考论文（PDF），并用模板分析其写作思路
   AI 会帮你分析这些论文的结构和技巧

📂 文献笔记/
   放你读过的论文笔记（.md），用模板写
   这是 AI 写论文时引用文献的唯一来源——不凭空编造引用

📂 素材/
   放实验数据、系统截图、流程图等原始材料
```

问：「以上目录用途清楚了吗？可以开始放入材料了。」

---

## Step 7: 检测内容并确认

等待用户说"好了"或"继续"后，检查工作区各目录：

- 检测 `文献笔记/` → 列出已放入的文件数量
- 检测 `论文模板/` → 是否存在 `.docx` 模板
- 检测 `素材/` → 是否有数据或图片

对每个**空目录**（但没有文件是正常的，如初稿、修改稿）：
- 问：「{目录名} 还没有内容，是忘记放还是没有/不需要？」
- 用户回答「忘记」→ 标为待办
- 用户回答「不需要」→ 跳过

---

## Step 8: 创建状态文件

在 `{workspace_path}/论文项目/毕业论文/` 下创建 `.pipeline-state.json`：

```json
{
  "workspacePath": "{workspace_path}",
  "major": null,
  "paperType": null,
  "wordCount": null,
  "setupComplete": true,
  "currentPhase": 0
}
```

---

## Step 9: 输出就绪报告

输出一份总结：

```
┌──────────────────────────────────────────┐
│        🎉 论文工作区就绪！              │
├──────────────────────────────────────────┤
│  工作区路径：{workspace_path}           │
│                                          │
│  已安装的依赖：xxx (N 个)               │
│  缺失的依赖：xxx (跳过/不安装)          │
│  已放入的文献笔记：N 篇                  │
│  论文模板：✅ / ❌                       │
│  素材：✅ / ⚠️ 待补充                    │
├──────────────────────────────────────────┤
│  下一步：运行 /论文流水线 开始写论文     │
└──────────────────────────────────────────┘
```

---

## 设计决策

- 所有模板文件名不包含"模板"二字（除原始模板文件本身），使用 `01-`、`02-` 前缀保持排序
- `.pipeline-state.json` 是 JSON 格式，后续 `/论文流水线` 会读取和更新它
- 如果用户中途退出，重新运行 `/启动论文工作区` 应检测现有目录并询问「检测到已有工作区，要重新初始化还是跳过？」
