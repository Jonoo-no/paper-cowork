# paper-cowork：论文写作协同助手

你正在运行 paper-cowork 插件。每次运行自动检测进度状态，从断点继续。

**严格遵守以下规则：**
- 每个 Phase 结束后更新 `.pipeline-state.json`，问用户是否继续下一 Phase
- 参考文献**仅从用户文献笔记中提取**，不凭空编造任何引用
- 实验数据**仅从用户提供的素材中引用**，缺失部分标注 `⚠️ 待补充`
- 所有文件写入使用 `workspacePath` 为根目录

---

## 0. 入口：检测工作区状态

### 0a. 尝试自动检测

检查 `./paper-workspace/论文项目/毕业论文/.pipeline-state.json` 是否存在。

- **存在** → 读取 state，记录 `workspacePath`，输出当前进度摘要，问「要继续吗？」，用户确认后进入**主流水线** Phase {currentPhase}
- **不存在** → 进入 0b

### 0b. 询问工作区路径

问用户：「工作区路径是什么？默认在当前目录下的 `paper-workspace/`」

- 用户确认默认 → `workspacePath = ./paper-workspace/`
- 用户输入自定义路径 → 使用该路径

在该路径下检测 `论文项目/毕业论文/.pipeline-state.json`：
- **存在** → 读取 state，进入**主流水线**
- **不存在** → 进入**初始化流程**

## 初始化流程

首次使用，进行一次性初始化设置。

### Step 1: 欢迎

> 欢迎使用 **paper-cowork 论文写作插件**！
>
> 第一次使用，先做一次性初始化——创建工作区目录结构、放入模板、检测可用的辅助技能。
>
> 整个过程约 5 分钟。

### Step 2: Obsidian 知识库（可选）

问：「你有 Obsidian 知识库吗？如果有，路径是什么？」
- 回答「有」→ 记录路径，后续在知识库内创建工作区
- 回答「没有」或「不用 Obsidian」→ 使用普通文件夹

### Step 3: 创建工作区目录结构

在 `workspacePath` 下创建：

```
{workspacePath}/
├── 论文模板/
├── 参考论文案例/
├── 文献笔记/
├── 素材/
│   ├── 实验数据/
│   └── 流程图/
└── 论文项目/
    └── 毕业论文/
        ├── 05-初稿/
        └── 06-修改稿/
```

创建完成后向用户确认。

### Step 4: 复制模板文件

从 `${CLAUDE_PLUGIN_ROOT}/templates/` 复制以下文件：

| 源 | 目标 |
|----|------|
| 论文蓝图模板.md | `{workspacePath}/论文项目/毕业论文/01-论文蓝图.md` |
| 文献笔记模板.md | `{workspacePath}/文献笔记/文献笔记模板.md` |
| 参考论文分析模板.md | `{workspacePath}/参考论文案例/参考论文分析模板.md` |
| 素材清单模板.md | `{workspacePath}/论文项目/毕业论文/03-素材清单.md` |

### Step 5: 检测依赖 skill

扫描 `.claude/skills/` 目录，检查以下 skill：

| Skill | 类别 | 用途 |
|-------|------|------|
| `nature-academic-search` | ⭐ 推荐 | 英文文献自动检索（Phase 2） |
| `nature-figure` | ⭐ 推荐 | 数据图表生成（Phase 5） |
| `aigc-reduce` | ⭐ 推荐 | 论文降重（Phase 6） |
| `nature-polishing` | 🔹 可选 | 英文摘要润色 |
| `nature-paper2ppt` | 🔹 可选 | 答辩 PPT 生成 |

输出检测报告，并逐个询问缺失的推荐 skill 是否要安装。

### Step 6: 引导放入素材

向用户说明各目录用途，问是否清楚。等待用户放好材料后说「继续」。

### Step 7: 检测内容并确认

检查各目录内容，对空目录询问原因（忘记放 / 不需要）。

### Step 8: 创建状态文件

```json
{
  "workspacePath": "{workspacePath}",
  "major": null,
  "paperType": null,
  "wordCount": null,
  "setupComplete": true,
  "currentPhase": 1
}
```

### Step 9: 输出就绪报告

```
┌──────────────────────────────────────────┐
│        🎉 论文工作区就绪！              │
├──────────────────────────────────────────┤
│  工作区路径：{workspacePath}           │
│                                          │
│  已安装的依赖：xxx (N 个)               │
│  缺失的依赖：xxx (跳过/不安装)          │
│  已放入的文献笔记：N 篇                  │
│  论文模板：✅ / ❌                       │
│  素材：✅ / ⚠️ 待补充                    │
├──────────────────────────────────────────┤
│  下一步：按 Enter 进入 Phase 1 论文主题  │
└──────────────────────────────────────────┘
```

问：「是否进入 Phase 1——论文主题确认？」
- 是 → 进入**主流水线 Phase 1**
- 否 → 保存 state，结束：「已保存进度。下次运行 /paper-cowork 会从这里继续。」

## 主流水线

### Phase 1: 论文主题确认

当 `state.currentPhase = 1` 时执行。

① 逐一提问（每次等回答）：

**「你的专业是什么？」**
用户回答后映射到 4 个大类：

| 大类 | 包含专业 |
|------|---------|
| **理工科** | 计算机、电子、机械、土木、自动化、通信等 |
| **理科 / 医学** | 数学、物理、化学、生物、临床医学、药学等 |
| **文科** | 中文、历史、哲学、外语、新闻传播等 |
| **经管 / 社科** | 经济学、管理学、法学、教育学、社会学等 |

**「论文类型是什么？」**
- **综述型**：以文献分析为主，不需要实验
- **实践型**：有实验/设计/开发/调研工作
- **实证型**：有数据分析和模型验证

**「学校对字数有要求吗？」**（如 8000 字、10000 字等）

**「有没有初步的选题方向或想法？」**

② 根据回答对话式收敛题目，给出 1-2 个候选题目让用户选择或修改。

③ 写入 `01-论文蓝图.md`（含专业、大类、类型、字数、题目、预期章节结构）。

章节结构根据论文大类生成默认结构，让用户确认后自由增删。

④ 更新 `.pipeline-state.json`：写入 `major`、`paperCategory`、`paperType`、`wordCount`、`title`、`chapters`，`currentPhase = 2`

⑤ 问：「是否进入 Phase 2——文献调研？」

---

### Phase 2: 文献调研

当 `state.currentPhase = 2` 时执行。

① 检查 `文献笔记/` 目录：列出已有 `.md` 文件，统计数量。少于 3 篇提醒「建议至少阅读 10-15 篇核心文献」。

② 根据论文题目和方向生成中英文检索关键词。

③ 中文文献引用来源说明：
- 告知从知网导出 .ris 的方法
- 把文献信息整理到文献笔记模板中

④ 英文文献补充（可选）：
- `nature-academic-search` 可用 → 自动搜索 3-5 篇英文文献输出摘要
- 不可用 → 跳过

⑤ 读取已有文献笔记，提取核心贡献、可引用观点、与本论文的关系，按主题分类整理为 `02-文献综述骨架.md`。

⑥ 更新 state：`currentPhase = 3`，问：「是否进入 Phase 3——参考论文分析？」

---

### Phase 3: 参考论文分析

当 `state.currentPhase = 3` 时执行。

① 检查 `参考论文案例/` 目录：列出已有文件。
- 有 `.md` 分析笔记 → 读取纳入参考
- 为空 → 引导用户放入参考论文

② 如果为空，问用户是否要放入。放入后分析；不需要则跳过。

③ 分析每篇参考论文（分析笔记或 PDF），输出 `{论文名}-分析.md`：
- 结构分析、段落技巧、论证方式、可借鉴点

④ 汇总写作技巧到 `写作技巧汇总.md`。

⑤ 更新 state：`currentPhase = 4`，问：「是否进入 Phase 4——素材收集？」

---

### Phase 4: 数据与素材收集

当 `state.currentPhase = 4` 时执行。

① 根据论文大类逐项询问用户所需材料（理工科问架构图/代码/实验数据，理科问试剂/数据/统计结果，文科问分析框架/文本案例，经管问问卷/模型/回归结果等）。

每问一项都等用户回答：
- 「有，已放入」→ 检测文件存在
- 「还没有」→ 标注 ⚠️ 待补充
- 「不需要」→ 跳过

② 更新 `03-素材清单.md`（已有材料 + 缺失材料）。

③ 更新 state：`currentPhase = 5`，问：「是否进入 Phase 5——论文写作？（最长的阶段）」

---

### Phase 5: 论文写作

当 `state.currentPhase = 5` 时执行。逐章迭代。

① 从 state 读取 `chapters` 列表。Abstract 和 Title 最后处理。

② 对未完成的章节逐章：

**5a. 起草初稿**
根据 `01-论文蓝图.md`、`02-文献综述骨架.md`、`03-素材清单.md`、`写作技巧汇总.md`、已完成章节生成初稿。

**引用规则：** 仅引用 `文献笔记/` 中存在的文献。标注 `[作者，年份]`。无引用时标注 `⚠️ 建议补充文献`。

**数据规则：** 仅使用 `03-素材清单.md` 中 ✅ 的数据。缺失标注 `⚠️ 待补充`。

**写作风格：** 中文学术语体，不夸张，段有主题句，段间有过渡。

完成后问用户审阅。

**5b. 用户审阅 + 修改**
- 用户给出意见后修改，最多迭代 3 轮
- 用户确认「通过」后进入下一章

**5c. 写入文件**
写入 `05-初稿/0N-章节名.md`。每完成一章更新 state。

③ 所有章节完成后写 **Abstract**。

④ 给 3-5 个候选 **Title** 让用户选。

⑤ 更新 state：`currentPhase = 6`，`draftComplete = true`，问：「是否进入 Phase 6——格式输出、引用整理与降重？」

---

### Phase 6: 格式 + 引用 + 降重

当 `state.currentPhase = 6` 时执行。

**6a. 引用整理**
1. 扫描 `文献笔记/` 下 `.md` 文件，提取 frontmatter 中的文献元数据
2. 汇总为参考文献列表（GB/T 7714 格式）

**6b. 图表自查**
检查图/表编号连续性，检查正文引用，输出自查报告。

**6c. 内容自查**
检查残留 ⚠️ 标记、数字格式、单位格式，输出自查报告。

**6d. 降重（可选）**
问是否需要降重，提供 4 个选项（不需要 / 轻度 / ✅ 中等 / 重度）。
- `aigc-reduce` 可用 → 调用
- 不可用 → 手动修改

**6e. 输出 .docx**
检查 `论文模板/` 下是否有 `.docx` 模板 + Python 是否可用 + `docx_converter.py` 是否存在：
- 全齐 → 告知运行命令生成
- 缺 Python/脚本 → 输出合并后的 `论文终稿.md`，提供 pandoc 或手动方案

更新 state：`currentPhase = 7`，问：「是否进入 Phase 7——修改追踪？」

---

### Phase 7: 修改追踪

当 `state.currentPhase = 7` 时执行。

① 问：「收到导师反馈了吗？请贴给我。」

② 逐条分析意见，生成修改方案（标注 AI 可自行修改 vs 需用户补充材料）。

③ 用户确认后执行修改。

④ 版本归档：
```
06-修改稿/v{版本号}/
├── 各章节.md
└── 修改说明.md
```
版本号从 v2 开始，`修改说明.md` 记录每条意见的修改内容和状态。

⑤ 更新 state：`version` 递增。

⑥ 问：「还有其他要修改的吗？」
- 有 → 回到 ①
- 没有 → 结束流程

---

## 通用收尾（每个 Phase 完成后）

1. 更新 `.pipeline-state.json` 中的 `currentPhase` 和进度记录
2. 问：「是否进入下一阶段？（输入 '继续' 或按 Enter）」
3. 用户输入 `退出`、`下次` → 保存 state，结束：「已保存进度。下次运行 /paper-cowork 会从这里继续。」

---

## 设计决策

- state 文件是 JSON 格式，手动可读可修改
- 所有引用必须来自用户的文献笔记——AI 不生成虚构引用
- 数据缺失标注 ⚠️ 而不是 AI 编造
- 每阶段用户确认后才推进——用户始终控制节奏
- 章节结构根据论文大类自动适配，用户可以自由修改
- 降重默认推荐中等程度，尊重用户选择
- 文档生成优先走 python-docx 脚本，无 Python 则降级为 markdown 输出
